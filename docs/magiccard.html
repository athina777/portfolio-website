How to Play

Pick one secret number from 1 to 63. For each card, check whether your number appears on that card. The game will add the card values together and reveal your number.

Card values: 1, 2, 4, 8, 16, 32

Choose the Cards That Contain Your Number

Your Result

Select the cards that include your number, then click Calculate.

Reflection

While creating this game, I learned how JavaScript can take user choices and turn them into a real calculation instantly on the page. The biggest challenge was making sure each card displayed the correct set of numbers based on its value and making the interface easy to follow. I also had to carefully test the logic with different secret numbers to confirm the total always matched the user's choice. Another challenge was designing clear feedback messages so users understand what to do before and after calculation. This project helped me see that HTML structures the content, CSS controls the visual design, and JavaScript adds behavior and decision-making. JavaScript feels different because it can react to events, run logic, and dynamically update results without refreshing the page.

JavaScript Code Walkthrough

The purpose of this game is to guess the number a user is thinking of between 1 and 63 by using six binary-value cards. The user interacts with the page by selecting Yes or No for each card, depending on whether their number appears in that card's list. I used variables such as cardValues to store each card's value and references like cardsContainer, gameOutput, and selectionStatus to control content on the page. Functions are used to build card numbers, create each card element, read selected cards, and calculate the final number. Event handlers respond to submit for calculation, click for reset, and change for live status updates as selections are made. The selected cards are processed by checking which radio inputs are set to Yes and collecting their matching card values. Those values are added together to compute the final result, and the answer is displayed dynamically in the result area without reloading the page. Conditional logic with if/else and switch determines validation messages, progress feedback, and when to show the final number.
